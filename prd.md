# 我的冰箱有什么｜产品需求文档

## 1. 产品目标

提供一个可在电脑和手机随时使用的冰箱物品管理服务。每位用户使用自己的账号登录；用户可拥有个人冰箱，也可加入家庭共享冰箱，和家人查看及维护同一份物品清单。

## 2. 核心使用场景

- 用户在电脑或手机登录自己的账号，查看个人冰箱。
- 用户创建家庭冰箱，通过邀请码链接邀请家人。
- 家人使用各自账号打开邀请链接并加入后，查看和编辑同一家庭冰箱的数据。
- 任一成员在电脑或手机增删改物品，其他成员刷新或重新打开后看到同步结果。
- 电脑关机后，手机仍可正常使用。

## 2.1 当前发布版本

- 网页发布在 GitHub Pages：<https://statjenny.github.io/fridge/>。
- 账户、冰箱和物品数据使用 Supabase 保存。
- 电脑和手机直接通过 HTTPS 访问网页，无需运行本地 `start.command`。
- 用户需完成邮箱注册/验证后登录；同一账号在不同设备上访问同一份云端数据。

## 3. 账户、冰箱与成员

### 3.1 账户

- 每位用户拥有独立账号和密码。
- 用户登录后只能访问自己拥有或已加入的冰箱。
- 新用户注册后自动创建一个仅自己可见的“我的冰箱”。

### 3.2 冰箱

- 一个用户可拥有多个冰箱，例如“我的冰箱”“父母家”。
- 一个冰箱可有多名成员。
- 用户可在个人冰箱和已加入的家庭冰箱之间切换。

### 3.3 成员角色

| 角色 | 权限 |
| --- | --- |
| 管理员 | 编辑物品、创建/撤销邀请链接、移除成员、管理冰箱名称 |
| 成员 | 查看、添加、编辑和删除该冰箱中的物品 |

创建冰箱的用户默认是管理员。

## 4. 邀请链接

- 管理员可为指定家庭冰箱创建邀请码链接。
- 链接使用不可预测的随机令牌，不使用递增编号或冰箱名称。
- 接收者须先登录自己的账号，再通过链接加入该冰箱。
- 管理员可设置链接有效期；默认有效期为 7 天。
- 链接可被管理员随时撤销；已撤销或过期链接不可再使用。
- 链接默认仅可成功加入一次。需要邀请多人时，管理员应分别创建链接。
- 已加入的成员不会因链接到期或被撤销而自动退出；管理员可单独移除成员。

## 5. 物品管理

### 5.1 物品字段

| 字段 | 必填 | 说明 |
| --- | --- | --- |
| 大类 | 是 | 冷冻、冷藏、干货、罐头、速食、零食之一 |
| 小类 | 是 | 由大类决定的可选项 |
| 物品名称 | 是 | 不允许为空 |
| 数量 | 是 | 正整数，默认 1 |
| 时间 | 是 | 默认当天日期 |
| 到期时间 | 否 | 用于后续到期提醒 |

### 5.2 分类

| 大类 | 小类 |
| --- | --- |
| 冷冻 | 海鲜、肉、主食、其他 |
| 冷藏 | 水果、蔬菜、饮料、酱料、咸菜、其他 |
| 干货 | 坚果、罐头、蔬菜干、其他 |
| 罐头 | 其他 |
| 速食 | 其他 |
| 零食 | 其他 |

### 5.3 操作与展示

- 用户点击“+ 添加物品”打开弹窗，录入物品信息。
- 物品按大类和小类分组，以紧凑表格显示名称、数量、到期时间。
- 点击物品名称后可编辑或删除；删除前必须二次确认。
- 系统记住当前用户、当前冰箱最近一次成功保存时使用的大类和小类，作为下次添加的默认值。
- 默认值保存在当前设备浏览器中；不同设备分别记忆，不作为需要同步的业务数据。
- 页面须适配电脑和手机屏幕。

## 6. 云端同步与安全

- 数据、账户和成员关系保存至 Supabase 云端数据库，不依赖电脑本地的 `items.json`。
- 静态网页通过 GitHub Pages 提供，地址为 <https://statjenny.github.io/fridge/>。
- 前端通过 HTTPS 访问云端服务；不再要求电脑启动本地服务、配置局域网地址或开放本机防火墙。
- 任何读取或写入请求都必须验证登录身份。
- 数据访问必须按“冰箱成员关系”授权：非成员不可读取或修改该冰箱的数据。
- 不得在浏览器中暴露云端管理员密钥。
- 本地已有的 `data/items.json` 应提供一次性导入到用户选定冰箱的能力。

### 6.1 前端安全基线

- 网页使用 Content Security Policy，仅允许本站、指定 Supabase 项目和固定版本的 Supabase 浏览器库。
- 第三方 Supabase 浏览器库必须固定精确版本并使用 Subresource Integrity 校验，避免运行时拉取浮动版本。
- 页面设置 `no-referrer`，邀请码链接不得作为 Referer 发送到其他站点。
- 浏览器端仅可使用 Supabase Publishable key；严禁提交或暴露 `service_role`、数据库密码和其他管理员密钥。

### 6.2 数据库安全基线

- 所有公开数据表启用 Row Level Security。
- 物品、冰箱和成员数据仅允许对应冰箱的已登录成员访问。
- 邀请与成员检查函数使用 `SECURITY DEFINER` 时，必须设置空 `search_path` 并使用完整对象名称。
- 上述函数必须撤销 `public` 的默认执行权限，仅向 `authenticated` 授予必要的执行权限。
- 当前项目已执行 `supabase-security-migration.sql` 以应用这些函数权限限制。

### 6.3 使用统计与隐私

- 网页接入 Google Analytics 4，Measurement ID 为 `G-EWZT9328LS`。
- 统计范围包括用户数、访问次数、页面浏览、活跃用户、访问地区、设备、来源渠道和功能事件。
- 功能事件包括：注册、登录、添加/编辑/删除物品、创建邀请和接受邀请。
- 统计事件不得发送邮箱、物品名称、冰箱名称、邀请码、Supabase 用户 ID 或其他可直接识别个人的信息。
- Google Analytics 提供国家、地区、城市等聚合地理信息；本产品不收集、展示或保存可访问的原始 IP 地址。
- 应在网站隐私说明中告知用户使用了 GA4；面向需要同意机制的地区时，应先获得分析 Cookie 同意。
- Realtime 报告通常在数分钟内显示活动；完整报表、地区和活跃用户数据可能需要 24–48 小时处理。

## 7. 界面要求

- 简洁、清爽、偏移动端清单 App 风格。
- 主页面突出当前冰箱、物品列表和添加入口。
- 提供冰箱切换入口，以及家庭成员和邀请链接管理入口（仅管理员可见）。
- 添加、编辑和删除使用清晰的弹窗流程。

## 8. 验收标准

1. 用户可在电脑和手机使用同一账号登录，并看到相同的个人冰箱数据。
2. 用户创建家庭冰箱后，可生成带有效期的一次性邀请码链接。
3. 另一位已登录用户打开有效链接后成功加入家庭冰箱。
4. 家庭成员在任一设备修改物品后，其他成员可获得同步后的数据。
5. 非成员、过期链接和已撤销链接均无法访问或加入该家庭冰箱。
6. 管理员可移除成员；被移除成员之后无法读取或修改该冰箱。
7. 电脑关闭时，手机仍可通过 HTTPS 网页正常使用。
8. 现有本地物品数据可导入一个指定冰箱，且不重复导入。
9. 浏览器仅能连接到允许的 Supabase 地址和固定版本的客户端库；邀请码链接不会作为 Referer 外泄。
10. 匿名请求无法执行创建邀请或接受邀请的数据库函数。
11. GA4 可记录页面访问及不含个人内容的功能事件；Realtime 可用于验证近期访问。

## 9. 当前未完成能力

- 在网页中创建新的家庭冰箱。
- 管理员撤销邀请链接、移除成员和修改冰箱名称。
- 将旧版本地 `data/items.json` 一次性导入云端冰箱。
- 邀请链接、成员管理和数据变更的实时推送通知。

## 10. 后续优化方向

- 已过期和即将到期的视觉提醒。
- 按类别、录入时间或到期时间排序与筛选。
- 批量删除、标记用完。
- 自定义分类和小类。
- 只读成员角色、通知与变更记录。
